02

createBrowserRouter — Route тохиргоо

Аппын route-уудыг нэг газарт бүртгэнэ. createBrowserRouter функц объект массив хүлээн авдаг.

router.jsx 
import { createBrowserRouter } from "react-router";
import ProductList   from "./ProductList";
import ProductDetail from "./ProductDetail";
import NotFound      from "./NotFound";

const router = createBrowserRouter([
  {
    path: "/products",
    Component: ProductList,          // /products → ProductList
  },
  {
    path: "/products/:id",           // :id → динамик параметр
    Component: ProductDetail,
  },
  {
    path: "*",                        // тохирохгүй бол 404
    Component: NotFound,
  },
]);

export default router;
main.jsx 
import { StrictMode }    from "react";
import { createRoot }   from "react-dom/client";
import { RouterProvider } from "react-router";
import router            from "./router";

createRoot(document.getElementById("root")).render(
  <StrictMode>
    <RouterProvider router={router} />   // router-ийг энд залгана
  </StrictMode>
);
📦

createBrowserRouter

Route-уудыг объект хэлбэрээр тодорхойлно. HTML5 History API ашиглана.

🔌

RouterProvider

Router-ийг бүх компонентод хүргэдэг "цахилгааны хайрцаг". Нэг л удаа main.jsx-д тавина.

🏷️

Component: vs element:

Component: ProductDetail гэж бичнэ — JSX биш, классын нэр шууд дамжуулна.

03

Хуудас хооронд шилжих

React Router шилжихэд хуудас дахин ачаалдаггүй — зөвхөн компонент солигдоно. Хоёр аргаар шилжиж болно.

<Link>
HTML холбоос шиг ашиглана. Хамгийн энгийн арга.
to="/products/42"
useNavigate()
Товч дарах, нөхцөл шалгасны дараа шилжих.
navigate("/products/42")
ProductList.jsx 
import { Link, useNavigate } from "react-router";

const PRODUCTS = [
  { id: 1, name: "Зурагт",              price: "890,000₮" },
  { id: 2, name: "Утас",                price: "1,200,000₮" },
  { id: 3, name: "Зөөврийн компьютер", price: "2,500,000₮" },
];

export default function ProductList() {
  const navigate = useNavigate();

  return (
    <div>
      <h1>Бүтэгдэхүүнүүд</h1>

      {PRODUCTS.map((p) => (
        <div key={p.id}>
          {/* Арга 1: Link — HTML <a> шиг боловч хуудас дахин ачаалдаггүй */}
          <Link to={`/products/${p.id}`}>{p.name}</Link>

          {/* Арга 2: useNavigate — товч дарах гэх мэт event-т ашиглана */}
          <button onClick={() => navigate(`/products/${p.id}`)}>
            Дэлгэрэнгүй</button>
        </div>
      ))}
    </div>
  );
}
⚠️

Анхааруулга: Хуудас шилжихэд <a href="..."> хэзээ ч ашиглаж болохгүй — хуудсыг бүтнээр дахин ачаалдаг. Зөвхөн <Link> эсвэл navigate() ашиглана.

navigate() функцийн бусад ашиглалт:

navigate утгууд 
navigate("/products/42")    // шууд хаяг руу явах
navigate(-1)               // буцах (browser "Back" товч шиг)
navigate(1)                // дахин урагш явах
navigate("/login", { replace: true })  // history-д бичихгүйгээр шилжих

export default function ProductDetail() {

  // URL /products/99 байвал → { id: "99" }
  const { id } = useParams();

  const navigate = useNavigate();

  // id-г fetch-д ашиглана (5-р хэсэгт дэлгэрэнгүй)
  console.log("Харах бүтэгдэхүүний id:", id); // "42"

  return (
    <div>
      <button onClick={() => navigate(-1)}>← Буцах</button>
      <h1>Бүтэгдэхүүн #{id}</h1>

Серверээс өгөгдөл дуудах

URL-ийн id-г авч, API-аас өгөгдлийг useEffect ашиглан дуудна. Fetch-ийн гурван төлөвийг заавал шалгах хэрэгтэй.

ProductDetail.jsx — бүрэн хувилбар 
import { useEffect, useState } from "react";
import { useParams, useNavigate } from "react-router";

export default function ProductDetail() {
  const { id }      = useParams();
  const navigate   = useNavigate();

  // Гурван state: өгөгдөл, ачаалалт, алдаа
  const [product, setProduct] = useState(null);
  const [loading, setLoading] = useState(true);
  const [error,   setError]   = useState(null);

  useEffect(() => {
    let cancelled = false;  // cleanup: хуучин request бохирдуулахаас хамгаална

    async function fetchProduct() {
      setLoading(true);
      setError(null);

      try {
        const res  = await fetch(`/api/products/${id}`);

        // HTTP алдаа шалгана (404, 500 гэх мэт)
        if (!res.ok) throw new Error(`Алдаа: ${res.status}`);

        const data = await res.json();
        if (!cancelled) setProduct(data);

      } catch (err) {
        if (!cancelled) setError(err.message);

      } finally {
        if (!cancelled) setLoading(false);
      }
    }

    fetchProduct();

    return () => { cancelled = true; };  // cleanup функц
  }, [id]);  // ← id өөрчлөгдөх бүрд дахин дуудна

  // Гурван төлөвийг заавал шалгах
  if (loading) return <p>Ачааллаж байна...</p>;
  if (error)   return <p>Алдаа: {error}</p>;
  if (!product) return <p>Олдсонгүй.</p>;

  return (
    <div>
      <button onClick={() => navigate(-1)}>← Буцах</button>
      <h1>{product.name}</h1>
      <p>Үнэ: {product.price}₮</p>
      <p>{product.description}</p>
    </div>
  );
}
⚠️

cancelled флаг яагаад хэрэгтэй вэ? Хэрэглэгч /products/1 рүү ороод шууд /products/2 рүү шилжвэл, 1-р request удаан ирж 2-р state-ийг дарж бичих тохиолдол гарч болно. cancelled = true тависнаар энэ "race condition"-оос хамгаалдаг.

💡

res.ok шалгах чухал! fetch() нь 404 эсвэл 500 хүлээн авсан ч алдаа шиддэггүй — зөвхөн сүлжээний алдаанд л exception гарна. Тиймд if (!res.ok) throw new Error(...) заавал бичих хэрэгтэй.
